Register module requirements

The Register module from top down is consisted of RegisterFile, Register, RegisterField, BitField, PhysicalRegister.

1.    PhysicalRegister class

This is conceptually what is really representing a register in the hardware.  But due to that we use the logical view Register class a lot more extensively, we use a shorter name for that class, but name the hardware register PhysicalRegister.

class PhysicalRegister : public Object {

public:

  virtual Object * Clone() const;

  virtual const std::string ToString() const;

  virtual const char* Type() const { return “PhysicalRegister”; }

  void Initialize(uint64 value, uint64 mask);

  bool IsInitialized(uint64 mask, bool* partial=NULL) const;

  uint64 Value(uint64 mask) const;

  void SetValue(uint64 value, uint64 mask);

  uint64 InitialValue(uint64 mask) const;

  bool HasAttribute(ERegisterAttribute);

  void SetAttribute(ERegisterAttribute):

  void ClearAttribute(ERegisterAttribute);

  const std::string& Name() const;

protected:

  std::string mName;

  uint64 mInitialValue;

  uint64 mValue;

  uint64 mMask;

  uint32 mSize;

  uint32 mAttributes; //!< Bit map of register attributes

};

The basic structure is as above, add more member variables or methods as necessary.

Clone method will return a copy constructed object of PhysicalRegister or its sub-type.

ToString method will return a string representation of the object.

Initialize will initial the whole or part of the PhysicalRegister, depending on the mask passed in.  This method need to check if we are initial the whole or part of the PhysicalRegister more than once, report an error if that’s the case.

IsInitialized check if the whole PhysicalRegister or part of it (depending on the mask) is initialized, the partial parameter, if the pointer is provided, will be set to true if the register content was partially initialized.

Value method returns the current value of the whole PhysicalRegister or part of it (depending on the mask).  Check should be done that the PhysicalRegister is initialized using the given mask, otherwise it should report an error.

InitialValue method returns the initial value of the whole PhysicalRegister or part of it (depending on the mask).  Check should be done that the PhysicalRegister is initialized using the given mask, otherwise it should report an error.

SetValue method set a value to the whole PhysicalRegister or part of it (depending on the mask).  Check should be done that the PhysicalRegister is initialized using the given mask, otherwise it should report an error.

mAttribute member variable contains a bit map of the multiple attributes, such as HasValue, Read, Write, etc.  These attributes can be defined as bit-map type enums.  Necessary accessor methods should be added to update the attributes.  By default, Read=1, Write=1 meaning allow both read and write access, HasValue=0.  After initialization or write then HasValue=1.

HasAttribute method return whether the register has a certain attribute or attributes set, it could be HasValue, Read, Write, or ReadWrite, etc.

SetAttribute and ClearAttribute set or clear one or more attributes.

The error report part use FAIL macro, it can be caught using EXPECT_FAIL macro in unit testing framework.

Enums can be added through the utilities/enum_classes/base_enum_classes.py utility, detailed information will be added to project guide.

2.    BitField class

BitField class represent contiguous bits in a RegisterField that come from the same PhysicalRegister.  A RegisterField could points to multiple PhysicalRegisters, this structure is supported with multiple BitFields in the RegisterField object that each points to different PhysicalRegisters.

class BitField : public Object {

  virtual Object * Clone() const;

  virtual const std::string ToString() const;

  virtual const char* Type() const { return “BitField”; }

  void InitializeField(uint64 value);

  void Initialize(uint64 value);

  bool IsInitialized(bool *partial=NULL) const;

  uint64 Value() const;

  uint64 InitialValue() const;

  uint64 FieldValue() const;

  uint64 InitialFieldValue() const;

  void SetFieldValue(uint64 value);

  void SetValue(uint64 value);

  bool HasAttribute(ERegisterAttribute);

  void SetAttribute(ERegisterAttribute):

  void ClearAttribute(ERegisterAttribute);

protected:

  PhysicalRegister* mRegister;

  uint64 mMask;

  uint32 mSize;

  uint32 mShift;

  uint32 mLsb;

};

BitField served as a translation from logical Register view to the bits locations in the actual PhysicalRegister.  Therefore a BitField object does not hold register value, it has a pointer to its associated PhysicalRegister object which hold the field value.  The pointer should be setup after the register files are already loaded.

The mMask member variable is used to pass on the initialization to the PhysicalRegister.  The bit-field value need to be shift left

The mShift member variable indicates the LSB of the field in the associated PhysicalRegister.  The mLsb member variable indicates the LSB of the field in the logical Register.  Normally mShift and mLsb is the same.  If in the register XML file the lsb attribute is omitted, which is the most cases, then mLsb will be set to the same as mShift.  If the lsb attribute is provided for the BitField in the register XML file, then the mLsb variable should be set to that specified value.

InitializeField method, the passed in value has been shifted to LSB=0, don’t need to apply mLsb shift.

Initialize method, the passed in value with LSB=mLsb, need to apply mLsb shift.

Similarly FieldValue method return a not yet shifted value with LSB=0, Value method return a shifted value with LSB=mLsb.

Similarly SetFieldValue takes a field value that has been shifted to LSB=0.  SetValue takes a value with LSB=mLsb.

Similarly InitialFieldValue method return a not yet shifted initial value with LSB=0, InitialValue method return a shifted value with LSB=mLsb.

3.    RegisterField class

RegisterField class represent bits in a logical Register that has a common name that come from the same PhysicalRegister.  A RegisterField could points to multiple PhysicalRegisters, this structure is supported with multiple BitFields in the RegisterField object that each points to different PhysicalRegisters, supporting class of that kind of RegisterField can be added when needed.

class RegisterField : public Object {

  virtual Object * Clone() const;

  virtual const std::string ToString() const;

  virtual const char* Type() const { return “RegisterField”; }

  void InitializeField(uint64 value);

  void Initialize(uint64 value);

  bool IsInitialized(bool *partial=NULL) const;

  uint64 Value() const;

  uint64 InitialValue() const;

  uint64 FieldValue() const;

  uint64 InitialFieldValue() const;

  void SetFieldValue(uint64 value);

  void SetValue(uint64 value);

  bool HasAttribute(ERegisterAttribute);

  void SetAttribute(ERegisterAttribute):

  void ClearAttribute(ERegisterAttribute);

  virtual void Setup(RegisterFile * registerFile);

  const std::string& Name() const;

protected:

  std::string mPhysicalRegisterName;

  std::string mName;

  uint32 mSize;

  std::vector<BitField *> mBitFields;

};

RegisterField class is very similar to BitField class, only that BitField class is continuous bits, and RegisterField can contain one or multiple BitField objects.  The Initialize method is useful when we initialize the top level logical Register with a full value, it can be passed down to all BitFields and each BitField take what they need by using their own masks.

RegisterField has an special Setup method, currently the main purpose is to obtain the PhysicalRegister pointer from RegisterFile by name lookup and pass down to all BitFields.  This is necessary since we will be cloning the whole RegisterField from an initial PE/Thread to all other subsequent threads, and BitFields in different PE/Threads obviously cannot point to the same PhysicalRegister in most of the cases, therefore some special setup is needed.

4.    Register class

Register class represent the logical view of a register to the ISG.

class Register : public Object {

  virtual Object * Clone() const;

  virtual const std::string ToString() const;

  virtual const char* Type() const { return “Register”; }

  void Initialize(uint64 value);

  bool IsInitialized(bool *partial=NULL) const;

  uint64 Value() const;

  uint64 InitialValue() const;

  void SetValue(uint64 value);

  bool HasAttribute(ERegisterAttribute);

  void SetAttribute(ERegisterAttribute):

  void ClearAttribute(ERegisterAttribute);

  virtual void Setup(RegisterFile * registerFile);

  const RegisterField* RegisterField(const std::string& name) const;

  RegisterField* RegisterField(const std::string& name);

  bool Boot() const;

  void InitializeRandomly();

  const std::string& Name() const;

protected:

  std::string mName;

  uint32 mSize;

  bool mBoot;

  std::vector<RegisterField *> mRegisterFields;

};

A Register instance might contain one or more RegisterField objects.  Most of the methods are simply distributing down to RegisterFields or gather up from RegisterFields.

The RegisterField methods provide easy ways to get a RegisterField of a Register to operate on, if the RegisterField object cannot be found, an error should be reported.  For example:

sctlr_reg->RegisterField(“M”)->Initialize(1); // okay

sctlr_reg->RegisterField(“XYZ”)->Value(); // fail

The mBoot attribute indicates whether we should generate boot code sequence to initialize this register.  This should be at logical Register level.

The InitializeRandomly is the basic form of initialization, just randomizing the whole register randomly.  More randomization methods will be added when related infrastructures are implemented.

5.    RegisterFile class

All the registers in one PE is contained in a RegisterFile class.

class RegisterFile : public Object {

  virtual Object * Clone() const;

  virtual const std::string ToString() const;

  virtual const std::string& Type() const { return “RegisterFile”; }

  virtual void Setup(RegisterFile * registerFile);

  const Register* Register(const std::string& name) const;

  RegisterField* Register(const std::string& name);

  const PhysicalRegister* PhysicalRegister(std::string& name) const;

  PhysicalRegister* PhysicalRegister(std::string& name) const;

  const std::string& Name() const;

protected:

  std::string mName;

  std::map<std::string, Register*> mRegisters;

  std::map<std::string, PhysicalRegister*> mPhysicalRegisters;

};

The RegisterFile class mainly served as a container of all the Register and PhysicalRegister objects in the PE/Thread, lookup a Register or PhysicalRegister is the most active use case of this class.  If a Register or PhysicalRegister with the given name cannot be found, an error should be reported.

6. Register file parsing.

Register files will be generated with register-builder scripts.  The parsing of register files can follow the way we parse instruction files.

XML syntax can be added to Force/config/force.config file.  Notice there might be multiple register files.  For default classes, we can add XML syntax like the following:

  <default_class category=”Object” type=”bit_field” class=”BitField”/>

We will soon add an ObjectRegistry so that various object can be cloned from registered objects, the object template can be found using the class names.

The register files can be loaded into Config class.  In ways similar to the handling of instruction files in Config.cc.

The XML parser can be implemented similar to InstructionParser in InstructionSet.cc.
