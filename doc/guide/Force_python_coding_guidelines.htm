Force/Python Coding Guidelines

1.       Coding guidelines

·         For modules and class names use upper case letters for word separators and to start a class name, and lower case otherwise. Same for method names and method parameters, except that first character should be lower case.

·         Method parameter (argument) names should be prefixed with a.

·         Class variable names should be prefixed with c.

·         Public/private class instance variable names should be prefixed with m.

·         Private class member names should be prefixed with a single underscore.

·         In class definitions, public methods should precede private methods.

·         For local variable names use lower case, with underscore as a separator between each word composing the variable name.

·         For filenames, use same naming convention as modules and classes. The file extension is .py.

·         For Python scripts, ie, python programs that can be run from the command line, use all lower case, with underscore as separator.  The file extension should be .py.

·         Constants may be defined at the module or class level, and if defined, should be all caps with underscores.

·         Two blank lines between each class definition. One blank line between each class method.

·         Do not use tabs, use spaces instead. Indent using four spaces.

·         Each class definition to be preceded with a doxygen-style header that includes a summary and details paragraph(s) for the class.

 

## @package pyexample

# Documentation for this module

#

 

## Documentation for this function

#

def func():

   pass

 

·         Each function/class-method definition should include a header summarizing the function/method, and describing inputs and outputs.

·         Include TODOs, caveats, limitations for classes/methods.

 

2.       Summary – naming conventions

 

Item

Description

Example

Class name

Camel-case, ie, upper case letters for word separators, lower case otherwise. Start with upper case.

class FooBarBaz(object):

Method name

Camel-case. Start with lower case.

def systemReset(self):

Method parameter

Camel-case, but starts with a. First letter after a should be upper case.

def stopBit(self,aBitPosition):

Class variable names

Prefix with lower case c.

Class GoodClass(object):

   cVarDef = 32

 

Public/private class instance variables

Prefix with lower case m.

def foo(self):

   mBitPosition = 0

Private class members

Private class member names should be preceded with a single underscore.

 

class Doodad(object):

   def _dingbat(self):

       pass

 

Local variables

Use lower case, with underscore as a separator between each word composing the variable name.

def wingding():

  wingding_state = []

File names

Use same naming convention as modules and classes. The file extension is .py

 

Exceptions.py

Script names

Use lower case, with underscore as a separator between each word composing the variable name. The file extension is .py.

master_run.py

Constants

All caps, with underscores as separators

CONST_EXC_NAME = “Synchronous exception”

 

 

3.       Other

For those Force developers who also use .vim as their preferred text editor, may want to use a vim plugin to maintain consistent spacing in C++ (two spaces for indentation) and Python (four spaces). To do so, create a file named python.vim that includes the following two lines:

 

                setlocal shiftwidth=4

     setlocal softtabstop=4

 

Save the file in your home directory as: ~/.vim/after/ftplugin/python.vim.
